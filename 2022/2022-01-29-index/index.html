索引

作者： qingling0910 时间： January 29, 2022 分类：

一、索引的类型
mysql一共有5种索引，如下所示：

唯一索引(UNIQUE)
主键索引(PRIMARY)
普通索引(NORMAL)
联合索引()
全文索引(FULLTEXT)

二、索引命名规则
主键索引名为pk_字段名；【强制】 —-pk 即primary key
唯一索引为uk_字段名； —- unique key
普通索引为idx_字段名 —- index的简称

三、创建索引
普通索引
1）建立索引
CREATE INDEX 索引名 ON 表名(字段名); 或
ALTER TABLE 表名 ADD INDEX 索引名(字段名);

2、唯一索引
1）建立索引 CREATE UNIQUE INDEX 索引名 ON 表名(字段名); 或
ALTER TABLE 表名 ADD UNIQUE INDEX 索引名(字段名);

3、主键索引
注意事项：
字段不能为null，不能重复；
表只能创建一个主键索引

1）创建索引 ALTER TABLE 表名 ADD PRIMARY KEY(字段名);

4、联合索引
1）创建索引
CREATE INDEX 索引名 ON 表名(字段名1,字段名2);

四、删除索引
DROP INDEX 索引名 ON 表名;

五、修改索引

ALTER TABLE 表名 RENAME INDEX 旧索引名 TO 新索引名;

六、查询索引
1）查询表中所有索引 show index from 表名;
show keys from 表名;

七、索引的优缺点
1、优点
1）索引可以加快查询速度，减少查询时间
2）唯一索引可以保证数据表中每一行的数据的唯一性

2、缺点
1）创建索引和维护索引要耗费时间
2）索引需要占用物理空间，除了数据表占用数据空间之外，每一个索引还要占用一定的物理空间
3）表中的数据进行增、删、改的时候，索引也要动态的维护；

八、索引对性能的影响
索引对于增删改操作的性能影响比较大
1、insert操作
insert的过程是，先把数据插入到表中，然后再把数据插入到相关索引中，如果这个表有5个索引，那么就得维护这5个索引，不管这个插入的数据是否为null值；
所以，索引个数越多，对于insert操作来说，维护的成本就越大，插入一条数据的速度也就越慢；
如果发现插入速度很慢，可以检查一下是否这个表的索引太多了；

把数据插入索引的过程中，为了维护索引中字段的顺序，会先在索引中查找这个值，如果能找到，就把这个值插到后面空闲的地方，如果没有找到，就先把值加入到叶子节点，然后在分支节点中新增这个值和指向叶子节点的指针（就是一个地址）

在这个过程中，如果某个页满了，还要申请一个空的页，把满的页拆分开，把一半的索引数据放到空闲页中，而且为了保证数据的一致性（这个插入操作是兵法的，可能有几十上百个线程同时进行），会给相关的索引加上闩锁（一种更低级的内存锁）

如此看来，这个过程的开销是很大的。

2、delete操作
delete操作刚好和insert相反，当删除一条数据时，会把这条数据涉及到的多个索引中的数据删除

如果分别在A表中的5个字段上建了索引，也就是5个索引，删除数据时，在各个索引中删掉，开销要比insert小；

3、update操作
这个操作不同于insert、delete，只有当update的这个字段，涉及到索引时，才需要维护索引，相对来说开销要小一些；
